Coderive is a code search engine that enables users to query codebases using much beyond conventional textual search. In particular, users can query codebases using 1) conventional textual search, 2) structure and semantics, 3) meaning, and 4) other.

1) Conventional textual search

Examples of conventional text-based query include:

 text[val[“type NewsSlice []*News”]]  (Go specific)

  => contains the text: “type NewsSlice []*News”

 text[val[“public static int cube(int x)”]]  (Java specific)

  => contains the text: “public static int cube(int x)”

 text[val[“\[.* for .* in .* for .* in .*\]”], regex[true]]  (Python specific)

  => contains any instance of a nested list comprehension

4) Other

“Other” includes any query that do not belong to the categories above, such as:

 lines[gte[1000]]

  => is at least 1000 lines of code


Indeed, these queries can be combined with each other, comma-separated (e.g. language[“Python”], class[field[name[“random_generator”]]], lines[ge[1000]]).
